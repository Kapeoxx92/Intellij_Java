Wymagany model danych: obiekt ma przechowywać elementy w wewnętrznej tablicy typu referencyjnego, np. Integer[], oraz przechowywać informację o aktualnej liczbie elementów. Nie używaj gotowej klasy Stack ani ArrayList.

To zadanie dotyczy również poprawnego zwalniania nieużywanych referencji. Po usunięciu elementu ze stosu slot tablicy, w którym ten element był przechowywany, powinien zostać wyczyszczony przez ustawienie wartości null. Dzięki temu usunięty obiekt nie pozostaje niepotrzebnie osiągalny.

W klasie SafeStack zaimplementuj metody:

int pop()
int peek()
int size()
boolean isEmpty()

Oczekiwane zachowanie:

PUSH x   - dodaje liczbę x na wierzch stosu
POP      - usuwa element z wierzchu i zwraca jego wartość
PEEK     - zwraca element z wierzchu bez usuwania
SIZE     - zwraca liczbę elementów przechowywanych na stosie
ISEMPTY  - zwraca true, gdy stos jest pusty, w przeciwnym razie false

Jeżeli operacja POP lub PEEK jest wykonywana na pustym stosie, program testujący wypisze "EmptyStackException". Nie musisz samodzielnie obsługiwać wypisywania komunikatu, ale Twoja klasa musi poprawnie rozpoznawać stan pustego stosu.

Możesz założyć, że stos może wymagać powiększania tablicy wewnętrznej.

Format wejścia:

linia 1: N - liczba operacji
kolejne N linii: jedna z operacji PUSH x, POP, PEEK, SIZE, ISEMPTY

Format wyjścia: dla każdej operacji POP, PEEK, SIZE i ISEMPTY program wypisuje wynik w osobnej linii.
